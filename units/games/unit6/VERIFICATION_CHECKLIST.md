### 2️⃣ 설정 화면 (Setup Screen)
- ✅ 모둠 수 설정 (2-8개)
- ✅ 모둠당 인원 설정 (1-6명)

- ✅ 던지기 횟수 설정 (1-5회)
- ✅ 난이도 선택 (쉬움/보통/어려움/전문가)

- ✅ 입력 검증 (최소/최대값)

### 3️⃣ 게임 화면 (Game Screen) - 타이밍 시스템
- ✅ 현재 모둠/플레이어 표시
- ✅ 라운드 정보 표시
- ✅ 콩 주머니 표시
- ✅ 훌라후프 표시
- ✅ 파워 게이지 표시

**3-1. 타이밍 게임 메커니즘**
- ✅ 첫 클릭: 파워 게이지 시작 (0%→100%→0% 자동 진동)
- ✅ 힌트 텍스트 변경: "콩 주머니를 클릭하여 시작!" → "적절한 타이밍에 다시 클릭!"
- ✅ 두 번째 클릭: 파워 게이지 정지 + 자동 던지기
- ✅ 게이지 색상: 빨강(약함) → 노랑 → 초록(최적 45-65%) → 노랑 → 빨강(강함)
- ✅ 최적 구간 마커 표시 (45%, 65%)

**3-2. 던지기 애니메이션**
- ✅ 포물선 궤적 애니메이션
- ✅ 콩 주머니 회전 효과
- ✅ 성공 시: 훌라후프 내부(중심 ±70% 반경) 착지
- ✅ 실패 시: 훌라후프 외부(반경 +20~80px) 착지
- ✅ 착지 후 1초 대기 → 다음 플레이어

**3-3. 확률 시스템**
- ✅ 파워값 기반 성공률 계산
  - 45-65%: 100% 성공률
  - 0-44%: 선형 감소 (0% → 98%)
  - 66-100%: 선형 감소 (98% → 0%)
- ✅ 난이도와 파워 성공률 결합
  - 쉬움: 기본 80% × 파워 성공률
  - 보통: 기본 60% × 파워 성공률
  - 어려움: 기본 40% × 파워 성공률
  - 전문가: 기본 20% × 파워 성공률
- ✅ 시각적 결과와 논리적 결과 일치

**3-4. 사운드 효과**
- ✅ 던지기 사운드 (스윕 효과)
- ✅ 성공 사운드 (C-E-G 코드)
- ✅ 실패 사운드 (하강 톤)
- ✅ Web Audio API 동작
- ⚠️ 사용자 제스처 후 활성화 (정상 동작)

**3-5. 점수판**
- ✅ 실시간 점수 업데이트
- ✅ 모둠별 색상 구분
- ✅ 현재 플레이 중인 모둠 강조
- ✅ 점수 증가 애니메이션

**3-6. 라운드 진행**
- ✅ 모든 모둠 완료 → 다음 라운드
- ✅ 라운드 카운터 업데이트
- ✅ 모든 라운드 완료 → 결과 화면 전환

**테스트 시나리오**:
```
1. 게임 시작
2. 콩 주머니 클릭 (1차) → 게이지 시작 확인
3. 최적 구간(초록색)에서 클릭 (2차) → 성공 확률 높음
4. 훌라후프 내부 착지 확인
5. 성공 사운드 재생 확인
6. 점수판 업데이트 확인
7. 여러 번 반복하여 실패 케이스도 확인
```

### 4️⃣ 결과 화면 (Result Screen)
- ✅ 순위 테이블 (금/은/동 메달)
- ✅ 모둠별 점수 표시
- ✅ 통계 정보 (1위, 최고점, 최저점, 평균)

**4-1. 단일 재사용 아이콘 시스템 (V3.1 Final 핵심)**
- ✅ 왼쪽 팔레트에 **1개** 콩 주머니 아이콘만 표시
- ✅ 아이콘 크기: 120px × 120px (대형)
- ✅ "드래그하세요" 라벨 표시
- ✅ 바운스 애니메이션 (위아래로 살랑살랑)
- ✅ 드래그 시작: `dragging` 클래스 추가 (반투명)
- ✅ 드래그 종료: 아이콘이 팔레트에 **그대로 유지됨**
- ✅ 무한 재사용: 같은 아이콘을 여러 번 드래그 가능
- ✅ `effectAllowed = 'copy'` 모드로 복사 동작

**4-2. 표 형식 그림그래프**
- ✅ 가로 행: 각 모둠별로 한 행씩
- ✅ 세로 열: 10, 20, 30, ... 눈금 (1칸 = 10개)
- ✅ 모둠 이름 셀: 모둠 색상 그라디언트
- ✅ 그래프 셀: 드래그 앤 드롭 영역
- ✅ 드래그 오버 시: 하이라이트 효과
- ✅ 드롭 시: 아이콘 채워짐 (분홍색 그라디언트)
- ✅ 클릭으로 삭제: 채워진 칸 클릭 → 비워짐
- ✅ 상단 눈금: 10, 20, 30, ... 자동 생성

**4-3. 그래프 그리기 과정**
```
예시: 희망 모둠 60개 성공

1. 왼쪽 아이콘 드래그 (1차) → 희망 행 첫 번째 칸에 드롭
2. 같은 아이콘 다시 드래그 (2차) → 두 번째 칸에 드롭
3. 반복 6번 → 6칸 채워짐 (60개)
4. "완성! 확인하기" 클릭
5. ✅ 60 = 60 (정확해요!) 피드백
```

**4-4. 정답 확인 시스템**
- ✅ 각 모둠별로 비교
  - 채운 칸 수 × 10 vs 실제 점수
- ✅ 정확한 경우: ✅ 모둠명: 60 = 60 (정확해요!)
- ✅ 틀린 경우: ❌ 모둠명: 50 ≠ 60 (다시 확인하세요)
- ✅ 모두 정확: 🎉 완벽해요! 메시지
- ✅ 하나라도 틀림: 💡 힌트 제공

**4-5. 추가 기능**
- ✅ "지우기" 버튼: 모든 그래프 초기화
- ✅ "다시하기" 버튼: 시작 화면으로 돌아가기
- ✅ 초기화 확인 다이얼로그

**테스트 시나리오**:
```
1. 결과 화면 진입
2. 왼쪽 팔레트에 1개 아이콘만 있는지 확인
3. 아이콘이 바운스 애니메이션 중인지 확인
4. 아이콘을 첫 번째 모둠 첫 번째 칸으로 드래그
5. 칸이 채워지고 아이콘이 팔레트에 그대로 있는지 확인
6. 같은 아이콘을 다시 드래그하여 두 번째 칸에 드롭
7. 여러 칸 채운 후 "완성! 확인하기" 클릭
8. 정답 피드백 확인
9. 잘못된 칸 클릭하여 삭제 확인
10. "지우기" 버튼으로 전체 초기화 확인
```
