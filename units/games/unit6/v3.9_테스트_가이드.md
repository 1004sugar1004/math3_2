# 🧪 v3.9 테스트 가이드

## 📋 테스트 체크리스트

### 1️⃣ 정교한 타이밍 시스템 테스트

#### 파워 게이지 55% (완벽 타이밍)
- [ ] 파워 게이지를 55%에 정확히 멈춤
- [ ] 성공 시 콩주머니가 훌라후프 **정중앙**에 떨어짐 확인
- [ ] 콘솔에 "✅ 성공! 파워:55% 정확도:100.0% 중심거리:<20px" 출력 확인
- [ ] 여러 번 반복해도 항상 중심 근처에 떨어지는지 확인

#### 파워 게이지 50~60% (매우 좋은 타이밍)
- [ ] 파워 게이지를 50%, 52%, 58%, 60%에 멈춤 (각각 테스트)
- [ ] 성공 시 콩주머니가 중심 근처에 떨어짐 확인
- [ ] 55%보다는 약간 떨어진 위치에 떨어지는지 확인
- [ ] 콘솔에 정확도 89~96% 출력 확인

#### 파워 게이지 45~65% (좋은 타이밍)
- [ ] 파워 게이지를 45%, 48%, 62%, 65%에 멈춤
- [ ] 성공 시 훌라후프 안쪽 (중심~가장자리)에 떨어짐 확인
- [ ] 콘솔에 정확도 78~80% 출력 확인

#### 파워 게이지 30~40% / 70~80% (보통)
- [ ] 파워 게이지를 30%, 40%, 70%, 80%에 멈춤
- [ ] 성공 시 가장자리에 떨어지거나, 실패 시 가까이 실패 확인
- [ ] 콘솔에 정확도 44~67% 출력 확인

#### 파워 게이지 0~20% / 90~100% (나쁨)
- [ ] 파워 게이지를 0%, 10%, 90%, 100%에 멈춤
- [ ] 대부분 실패하며, 훌라후프에서 멀리 떨어짐 확인
- [ ] 콘솔에 정확도 0~22% 출력 확인

### 2️⃣ 시각적 정확도 피드백 테스트

#### 성공 위치 비교
1. **55% 완벽 타이밍**: 
   - [ ] 정중앙 (반경의 0~25%)에 떨어짐
   - [ ] 매우 작은 범위 내에서만 변동

2. **50~60% 좋은 타이밍**:
   - [ ] 중심 근처 (반경의 0~35%)에 떨어짐
   - [ ] 55%보다 약간 넓은 범위

3. **45~65% 괜찮은 타이밍**:
   - [ ] 안쪽 (반경의 0~50%)에 떨어짐
   - [ ] 가장자리에도 떨어질 수 있음

4. **40% 이하 / 70% 이상**:
   - [ ] 가장자리 (반경의 0~85%)에 떨어지거나 실패
   - [ ] 실패 시 가까이 떨어짐 (+15~50px)

#### 실패 거리 비교
1. **45~65% (거의 성공할 뻔)**:
   - [ ] 훌라후프 바로 밖 (+15~30px)에 떨어짐
   - [ ] "아깝다!" 느낌

2. **30~40% / 70~80% (보통 실패)**:
   - [ ] 중간 거리 (+20~67px)에 떨어짐

3. **0~20% / 90~100% (완전 실패)**:
   - [ ] 멀리 (+40~80px)에 떨어짐
   - [ ] 명확히 실패한 느낌

### 7️⃣ 디버깅 로그 확인

#### 브라우저 개발자 도구 열기
- **PC**: F12 또는 우클릭 → 검사
- **모바일**: Safari/Chrome 원격 디버깅

#### 성공 로그 예시
```
✅ 성공! 파워:55% 정확도:100.0% 중심거리:12.3px (반경:85.0px)
✅ 성공! 파워:52% 정확도:93.3% 중심거리:28.7px (반경:85.0px)
✅ 성공! 파워:48% 정확도:86.7% 중심거리:45.2px (반경:85.0px)
```

#### 실패 로그 예시
```
❌ 실패! 파워:75% 정확도:55.6% 중심거리:134.5px (반경:100.0px)
❌ 실패! 파워:30% 정확도:44.4% 중심거리:158.9px (반경:100.0px)
❌ 실패! 파워:0% 정확도:0.0% 중심거리:189.3px (반경:100.0px)
```

#### 확인 사항
- [ ] 중심거리가 반경보다 작으면 → 성공 ✅
- [ ] 중심거리가 반경보다 크면 → 실패 ❌
- [ ] 정확도가 높을수록 → 중심거리 작음 (성공 시)
- [ ] 정확도가 낮을수록 → 중심거리 큼 (실패 시)

## 🎯 기대 결과

### ✅ 성공 기준

1. **정교한 타이밍**: 55%에 가까울수록 성공률 증가
2. **시각적 정확도**: 최적 타이밍 → 중심에 떨어짐
3. **반응형**: 모든 기기에서 정확한 히트 판정
4. **애니메이션**: 부드럽고 자연스러움
5. **디버깅**: 콘솔 로그로 정확도 확인 가능

### ❌ 실패 시 확인 사항

1. **콩주머니가 훌라후프 밖에 떨어지는데 성공 판정**:
   - 콘솔 로그 확인: 중심거리 < 반경?
   - 훌라후프 getBoundingClientRect() 확인

2. **55% 완벽 타이밍인데 가장자리에 떨어짐**:
   - accuracyRatio 계산 확인
   - maxDistanceRatio 값 확인

3. **모바일에서 위치가 부정확함**:
   - 화면 크기 스케일링 확인
   - failDistanceScale 값 확인

4. **애니메이션이 끊김**:
   - requestAnimationFrame 사용 확인
   - CSS transform 사용 확인
